How to use

Cloud-based (we host)

You can't send metrics from your apps into cloud MVP now. Current version of cloud MVP uses and shows only its own self-monitored metrics. It is made for providing instructions, explanations, how-to's and for demonstration of "Do They Use" service current abilities without any installations on your side.

Go to Analytics app (by clicking here or in menu above) to try it.

Self-hosted (you host)

You can see analytics from your app using self-hosted option. It is easy to setup. Below are detailed instructions on how to do it.

Zero coding demo

  1. Download your_app.html (on saving choose "Webpage, HTML only" option, not "Webpage, Complete" — because the file is self-sufficient);
  2. Download dtu_app.html (the same: on saving choose "Webpage, HTML only");
  3. Open both files in the same web browser;
  4. Interact with the elements of "your_app.html" (click links, change values);
  5. Refresh the page or click on "Refresh filter" button in "dtu_app.html" to see the stats from "your_app.html".

This is how it works in general.

In "Getting started" section below there is a detailed instruction on how to do the same but for your real app.

Getting started

  1. Save this dtu_app.html single-page HTML file of the Analytics app to any place. When saving choose "Webpage, HTML only" option, not "Webpage, Complete" because this HTML file has absolute paths for all the imports it needs. You can place "dtu_app.html" file anywhere and it will work from any place if that place is in the same origin with your app (file:// or http://domain or https://domain) in the same browser (because it uses window.localStorage of web browser to retrieve stats from your app to the Analytics app).
  2. Import and initialize JavaScript SDK in your web app. Put the following code right before </body> tag in your app:
    <script src="https://dotheyuse.com/src/db/dtu_db.js"></script>
<script src="https://dotheyuse.com/src/rx_api/dtu_rx_api.js"></script>

<script src="https://dotheyuse.com/src/customer/dtu_sdk_js/dtu_sdk.js"></script>
<script type="text/javascript">
const dtu = dotheyuse({
  'ctag': 'DEMO MVP',
  'topic': 'real usage'
});
</script>
  3. Tag with "data-dtu" attribute any (of JS SDK supported types) active elements in your web app. For example, tag a hyperlink:
    <a href="..." data-dtu="some link">...</a>
  4. Interact with that element. For example for the "link" (anchor) element JS SDK monitors "click" interaction. For other elements it could be "change" event type. Refer to JS SDK source code to find exact events.
  5. Open "dtu_app.html" file that you downloaded in step 1 and see that your interaction is already displayed in the app. Please note, that "dtu_app.html" has to be in the same origin with your app (for example, if you run your app on http://localhost:8000 host-port then "dtu_app.html" must be opened somewhere in the same http://localhost:8000/... host-port origin in the same web browser).

Furter steps

  • Tag more elements with "data-dtu" attribute to see them in monitoring stats;
  • Contribute to JS SDK if something not implemented in SDK yet, not working or you found a bug;
  • Provide your feedback — so we could to improve the product further.
