How to setup for your app

Cloud-based (we host)

You can't send metrics from your apps into cloud MVP right now. Current version of cloud MVP uses and shows only its own self-monitored metrics. It is made to provide you with instructions, explanations, how-to's and demonstration of the service current abilities without any setup efforts from your side.

Go to Analytics app to try it. It has both synthetic auto-generated (lite and heavy data) and your real current usage live data:

Self-hosted (you host)

You can see real analytics from your web app if you use self-hosted setup.

Below are detailed instructions on how you can do it. First 2 options (Zero coding demo and <iframe> demo) — are demos to show you how to attach analytics to sample "your_app.html". Third option (Enable analytics for your web app) — is a step-by-step guide on how to make the same for your real app.

Zero coding demo

  1. Download your_app.html (on saving choose "Webpage, HTML only" option, not "Webpage, Complete" — because the file is self-sufficient);
  2. Download dtu_app.html (the same: on saving choose "Webpage, HTML only");
  3. Open both files in the same web browser;
  4. Interact with the elements of "your_app.html" (click links, buttons, change values);
  5. Refresh "dtu_app.html" (or click on "5 min" link on the) page to see the stats from "your_app.html".

This is how it works in general.

<iframe> demo

Open <iframe> demo in a new tab, play with it and then return back. It is completely ready to play with. We'll wait for you here.

This is a demo page with <iframe>s for "your_app.html" (on the left) and "dtu_app.html" (on the right) — so you can see everything on a single page with no need of switching between browser tabs. This is how it looks:

This single-page <iframe> demo is available in 2 variants:

  1. Open iframe_app_local.html and save it locally (on saving choose "Webpage, HTML only" option) into the same directory with 2 files that you've already downloaded during steps 1 and 2 of "Zero coding demo" instruction above. This <iframe> demo file works with local copies of "your_app.html" and "dtu_app.html";
  2. Or alternatively, open iframe_app_remote.html and save it anywhere locally (on saving choose "Webpage, HTML only" option as well). This demo file works without local copies of "your_app.html" and "dtu_app.html" because it loads them into <iframe>s from dotheyuse.com site.

Also these 2 <iframe> demo files will work from dotheyuse.com website without any need to save them locally. You can play with them on dotheyuse.com as you've already done in the beginning. It will work.

Enable Do They Use analytics for your web app

Below is a detailed instruction on how to do the same analytics not for "your_app.html" but for your real web app.

  1. If you haven't saved yet, save dtu_app.html single-page HTML file of the DTU analytics app to any place. When saving choose "Webpage, HTML only" option, not "Webpage, Complete" because this HTML file has absolute paths for all the imports it needs. You can place "dtu_app.html" file anywhere and it will work from any place if that place is in the same origin with your app (file:// or http://domain or https://domain) in the same browser (because it uses window.localStorage of web browser to retrieve stats from your app to the Do They Use analytics app).
  2. Import and init JavaScript SDK in your web app: put the following code right before </body> tag in your web app:
    <script src="https://dotheyuse.com/src/db/dtu_db.js"></script>
<script src="https://dotheyuse.com/src/rx_api/dtu_rx_api.js"></script>

<script src="https://dotheyuse.com/src/customer/dtu_sdk_js/dtu_sdk.js"></script>
<script type="text/javascript">
  const dtu = dotheyuse({
    'ctag': 'DEMO MVP',
    'topic': 'real usage'
});
</script>
  3. Tag with "data-dtu" attribute any (of DTU JS SDK supported types) elements of your web app. For example, tag a hyperlink:
    <a href="..." data-dtu>...</a>
    or with custom "data-dtu" attribute value: 
    <a href="..." data-dtu="some link">...</a>
  4. Interact with that element. For example for the "link" (anchor) element JS SDK monitors "click" interaction. For other elements it could be "change" event type. Refer to JS SDK source code to find exact elements and event types.
  5. Open "dtu_app.html" file that you downloaded in step 1 and see that your interaction is already displayed in the app. Please note, that "dtu_app.html" has to be in the same origin with your app (for example, if you run your app on http://localhost:8000 host-port then "dtu_app.html" must be opened somewhere in the same http://localhost:8000 host-port origin in the same web browser).

Furter steps

  • Tag more elements in your web app with "data-dtu" attributes to see them in Do They Use analytics app;
  • Contribute to JS SDK repository (with code pull requests or by submitting issues and features requests) if something not implemented in JS SDK yet, not working or you found a bug;
  • Provide your feedback — so we could to improve this product further.
