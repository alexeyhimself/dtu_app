Installation of analytics app

To pull and run Docker container with DoTheyUse Product Ananlytics app use instructions at Docker Hub web page.

Installation of ANALYTICS SDK into your APP

  1. Import and init DTU JavaScript SDK in your web app: put the following code right before </body> tag in your web app:
    <!-- Use JS SDK hosted by us (default): -->
      <script src="https://dotheyuse.com/src/dtu_app/src/customer/dtu_sdk_js/dtu_sdk.js"></script>
      <!-- or use JS SDK hosted by your container: -->
      <!--script src="/src/customer/dtu_sdk_js/dtu_sdk.js"></script-->
      <script type="text/javascript">
        const dtu = dotheyuse({
          'topic': 'My sample app'
        });
      </script>
  2. Tag with "data-dtu" attribute any (of DTU JS SDK supported types) elements of your web app. For example, tag a hyperlink:
    <a href="..." data-dtu>...</a>
    or with custom "data-dtu" attribute value: 
    <a href="..." data-dtu="some link">...</a>

Using analytics app

  • Interact with tagged element(s) in your app. For example for the "link" (anchor) element JS SDK monitors "click" interaction. For other elements it could be "change" event type. Refer to JS SDK source code to find exact elements and event types.
  • Open "dtu_app.html" file that you downloaded in step 1 and see that your interaction is already displayed in the app. Please note, that "dtu_app.html" has to be in the same origin with your app (for example, if you run your app on http://localhost:8000 host-port then "dtu_app.html" must be opened somewhere in the same http://localhost:8000 host-port origin in the same web browser).

Furter steps

  • Tag more elements in your web app with "data-dtu" attributes to see them in Do They Use analytics app;
  • Contribute to JS SDK repository (with code pull requests or by submitting issues and features requests) if something not implemented in JS SDK yet, not working or you found a bug;
  • Provide your feedback — so we could to improve this product further.
